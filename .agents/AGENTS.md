# IRIS Project Rules

## Creative Bible Authority
The [docs/IRIS_CREATIVE_BIBLE.md](file:///c:/Users/hamza/.gemini/antigravity-ide/scratch/iris-studio-website/docs/IRIS_CREATIVE_BIBLE.md) is the highest creative and technical authority in this repository. Every future feature, redesign, animation, component, layout, asset, and interaction must align with the vision set forth in it.

### Rules of Engagement
1. **Adherence**: Read the relevant chapters from the Creative Bible before designing or implementing any code changes.
2. **Consistency Over Speed**: Protecting consistency across the entire project is more important than fast execution.
3. **Conflict Resolution**: If a user request or implementation plan conflicts with the philosophy documented in the Creative Bible, do not silently implement it. Explicitly highlight the conflict and ask the user for clarification.
4. **No Unplanned Extensions**: Never invent new design principles, change the brand philosophy, or create undocumented interaction patterns unless they are first formally updated in the Creative Bible.
